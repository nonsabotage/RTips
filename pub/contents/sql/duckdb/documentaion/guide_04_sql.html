2.1 Populating a Table with Rows

INSERTは行単位でデータを追加するときに使われる。

INSERT INTO weather
VALUES ('San Francisco', 46, 50, 0.25, '1994-11-27');

数値でないときにはシングルクォーテションで値をかこむ必要がある。

INSERT INTO weather
VALUES ('San Francisco', 46, 50, 0.25, '1994-11-27');

インサートするときにカラムの順序は異なっていても大丈夫である。

INSERT INTO weather (date, city, temp_hi, temp_lo)
VALUES ('1994-11-29', 'Hayward', 54, 37);

Many developers consider explicitly listing the columns better style than relying on the order implicitly.

バルクロードするときにはCOPYを使うことでより高速になる。

COPY weather
FROM 'weather.csv'

2.4 Aggregate Functions

  • count, sum, avg, max, min
Code 
SELECT max(temp_lo)
FROM weather;
1 records
max(temp_lo)
46

集約関数は次のようにWHERE句で使えない。 これは集約計算の対象がWHEREで絞った結果であるため。

-- これはエラーになる
SELECT city
FROM weather
WHERE temp_lo = max(temp_lo);     -- WRONG

WHERE句で使うにはスカラー値をクエリする必要がある。サブクエリにすることで独立したクエリとして計算が行える。。

Code 
SELECT city
FROM weather
WHERE temp_lo = (SELECT max(temp_lo) FROM weather);
1 records
city
San Francisco
Code 
SELECT city, max(temp_lo)
FROM weather
GROUP BY city;
2 records
city max(temp_lo)
Hayward 37
San Francisco 46

計算結果に対してフィルターすることが可能である。

Code 
SELECT city, max(temp_lo) as A
FROM weather
GROUP BY city
HAVING A < 40;
1 records
city A
Hayward 37
Code 
SELECT city, max(temp_lo)
FROM weather
WHERE city LIKE 'S%'            -- (1)
GROUP BY city
HAVING max(temp_lo) < 40;
0 records
city max(temp_lo)

HAVINGは常に集約値を使うことに注意する。

3 Widnow Functions

いわゆるWindow関数が使える

-- generate a "row_number" column with containing incremental identifiers for each row
SELECT row_number() OVER () FROM sales;
-- generate a "row_number" column, by order of time
SELECT row_number() OVER (ORDER BY time) FROM sales;
-- generate a "row_number" column, by order of time partitioned by region
SELECT row_number() OVER (PARTITION BY region ORDER BY time) FROM sales;
-- compute the difference between the current amount, and the previous amount,
-- by order of time
SELECT amount - lag(amount) OVER (ORDER BY time) FROM sales;
-- compute the percentage of the total amount of sales per region for each row
SELECT amount / sum(amount) OVER (PARTITION BY region) FROM sales;

3.3.2 Framing

PRECEDINGやFOLLOWINGを使うことで、相対的な位置関係を使うことが可能である。 この距離はROWSの整数としても指定することができる。 フレームのデフォルト値はUNBOUNDED PRECEDINGからCURRENT ROWである。 EXCLUDE句は使用するうと現在の行の周りの行をフレームから除外することができる。

RANGEフレーミングを行う。

Code 
SELECT "Plant", "Date",
    avg("MWh") OVER (
        PARTITION BY "Plant"
        ORDER BY "Date" ASC
        RANGE BETWEEN INTERVAL 3 DAYS PRECEDING
                  AND INTERVAL 3 DAYS FOLLOWING)
        AS "MWh 7-day Moving Average"
FROM "Generation History"
ORDER BY 1, 2;

SELECT "Plant", "Date",
    CASE

            PARTITION BY "Plant"

3.3.3 WINDOW Clauses

WINDOWを選言することでマクロのような使い方ができる。

Code 
SELECT "Plant", "Date",
    min("MWh") OVER seven AS "MWh 7-day Moving Minimum",
    avg("MWh") OVER seven AS "MWh 7-day Moving Average",
    max("MWh") OVER seven AS "MWh 7-day Moving Maximum",
    min("MWh") OVER three AS "MWh 3-day Moving Minimum",
    avg("MWh") OVER three AS "MWh 3-day Moving Average",
    max("MWh") OVER three AS "MWh 3-day Moving Maximum"
FROM "Generation History"
WINDOW
    seven AS (
        PARTITION BY "Plant"
        ORDER BY "Date" ASC
        RANGE BETWEEN INTERVAL 3 DAYS PRECEDING
                  AND INTERVAL 3 DAYS FOLLOWING),
    three AS (
        PARTITION BY "Plant"
        ORDER BY "Date" ASC
        RANGE BETWEEN INTERVAL 1 DAYS PRECEDING
        AND INTERVAL 1 DAYS FOLLOWING)
ORDER BY 1, 2;

## Aggregate Functions

- count, sum, avg, max, min


```{sql}
#| connection: con
SELECT max(temp_lo)
FROM weather;
```

集約関数は次のようにWHERE句で使えない。
これは集約計算の対象がWHEREで絞った結果であるため。

```sql
-- これはエラーになる
SELECT city
FROM weather
WHERE temp_lo = max(temp_lo);     -- WRONG
```

WHERE句で使うにはスカラー値をクエリする必要がある。サブクエリにすることで独立したクエリとして計算が行える。。


```{sql}
#| connection: con
SELECT city
FROM weather
WHERE temp_lo = (SELECT max(temp_lo) FROM weather);
```


```{sql}
#| connection: con
SELECT city, max(temp_lo)
FROM weather
GROUP BY city;
```


計算結果に対してフィルターすることが可能である。


```{sql}
#| connection: con
SELECT city, max(temp_lo) as A
FROM weather

HAVING A < 40;
```

```{sql}
#| connection: con
SELECT city, max(temp_lo)
FROM weather
WHERE city LIKE 'S%'            -- (1)
GROUP BY city
HAVING max(temp_lo) < 40;
```

`HAVING`は常に集約値を使うことに注意する。

# Widnow Functions

いわゆるWindow関数が使える




```sql
-- generate a "row_number" column with containing incremental identifiers for each row
SELECT row_number() OVER () FROM sales;
-- generate a "row_number" column, by order of time
SELECT row_number() OVER (ORDER BY time) FROM sales;
-- generate a "row_number" column, by order of time partitioned by region
SELECT row_number() OVER (PARTITION BY region ORDER BY time) FROM sales;
-- compute the difference between the current amount, and the previous amount,
-- by order of time
SELECT amount - lag(amount) OVER (ORDER BY time) FROM sales;
-- compute the percentage of the total amount of sales per region for each row
SELECT amount / sum(amount) OVER (PARTITION BY region) FROM sales;

```

```{sql}
#| connection: con
CREATE OR REPLACE TABLE penguins AS SELECT * FROM read_parquet('penguins.parquet')
```


```{sql}
#| connection: con
SELECT row_number() OVER () as row_number, * FROM penguins;
```

```{sql}
#| connection: con
SELECT row_number() OVER (ORDER BY bill_length_mm), bill_length_mm FROM penguins;
```

```{sql}
#| connection: con
SELECT row_number() OVER (PARTITION BY species, island ORDER BY bill_length_mm), species, island, bill_length_mm FROM penguins;
```

```{sql}
#| connection: con
SELECT bill_length_mm / AVG(bill_length_mm) OVER(PARTITION BY species, island) A,  
AVG(bill_length_mm) OVER(PARTITION BY species, island) B, 
species, island, bill_length_mm
FROM penguins;
```

## Evaluation

### Partition and Ordering

```{sql}
#| connection: con
CREATE OR REPLACE TABLE "Generation History" AS
  FROM 'data/csv/power-plant-generation-history.csv';
```

```{sql}
#| connection: con
SELECT * FROM "Generation History" LIMIT 5;
```

まずは`row_number`の挙動をみる。

```{sql}
#| connection: con
SELECT
    "Plant",
    "Date",
    row_number() OVER (PARTITION BY "Plant" ORDER  BY "Date") AS "Row"
FROM "Generation History"
ORDER BY 1, 2;
```

### Framing

`PRECEDING`や`FOLLOWING`を使うことで、相対的な位置関係を使うことが可能である。
この距離はROWSの整数としても指定することができる。
フレームのデフォルト値は`UNBOUNDED PRECEDING`から`CURRENT ROW`である。
`EXCLUDE`句は使用するうと現在の行の周りの行をフレームから除外することができる。

![](./images/framing-row.png)

`RANGE`フレーミングを行う。

```{sql}
#| connection: con
SELECT "Plant", "Date",
    avg("MWh") OVER (
        PARTITION BY "Plant"
        ORDER BY "Date" ASC
        RANGE BETWEEN INTERVAL 3 DAYS PRECEDING
                  AND INTERVAL 3 DAYS FOLLOWING)
        AS "MWh 7-day Moving Average"
FROM "Generation History"
ORDER BY 1, 2;
```

範囲がないときにNULLになるようにする。

```{sql}
#| connection: con
SELECT "Plant", "Date",
    CASE
        WHEN COUNT("MWh") OVER (
            PARTITION BY "Plant"
            ORDER BY "Date" ASC
            RANGE BETWEEN INTERVAL 3 DAYS PRECEDING
                      AND INTERVAL 3 DAYS FOLLOWING
        ) >= 7 THEN
            avg("MWh") OVER (
                PARTITION BY "Plant"
                ORDER BY "Date" ASC
                RANGE BETWEEN INTERVAL 3 DAYS PRECEDING
                          AND INTERVAL 3 DAYS FOLLOWING
            )
        ELSE
            NULL
    END AS "MWh 7-day Moving Average"
FROM "Generation History"
ORDER BY 1, 2;

```

### WINDOW Clauses

WINDOWを選言することでマクロのような使い方ができる。

```{sql}
#| connection: con
SELECT "Plant", "Date",
    min("MWh") OVER seven AS "MWh 7-day Moving Minimum",
    avg("MWh") OVER seven AS "MWh 7-day Moving Average",
    max("MWh") OVER seven AS "MWh 7-day Moving Maximum",
    min("MWh") OVER three AS "MWh 3-day Moving Minimum",
    avg("MWh") OVER three AS "MWh 3-day Moving Average",
    max("MWh") OVER three AS "MWh 3-day Moving Maximum"
FROM "Generation History"
WINDOW
    seven AS (
        PARTITION BY "Plant"
        ORDER BY "Date" ASC
        RANGE BETWEEN INTERVAL 3 DAYS PRECEDING
                  AND INTERVAL 3 DAYS FOLLOWING),
    three AS (
        PARTITION BY "Plant"
        ORDER BY "Date" ASC
        RANGE BETWEEN INTERVAL 1 DAYS PRECEDING
        AND INTERVAL 1 DAYS FOLLOWING)
ORDER BY 1, 2;
```

### Filtering the Results of Window Functions Using QUALIFY


```{sql}
#| connection: con
SELECT "Plant", "Date",
    min("MWh") OVER seven AS "MWh 7-day Moving Minimum",
    quantile_cont("MWh", [0.25, 0.5, 0.75]) OVER seven
        AS "MWh 7-day Moving IQR",
    max("MWh") OVER seven AS "MWh 7-day Moving Maximum",
FROM "Generation History"
WINDOW seven AS (
    PARTITION BY "Plant"
    ORDER BY "Date" ASC
    RANGE BETWEEN INTERVAL 3 DAYS PRECEDING
              AND INTERVAL 3 DAYS FOLLOWING)
ORDER BY 1, 2;
```
