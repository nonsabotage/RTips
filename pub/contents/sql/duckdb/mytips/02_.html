2 QUALIFY

WINDOW()関数の結果を使ったクエリができる。データの読込が１度であるため、相関サブクエリでフィルターしているときにはより効率的、高速になることがある。履歴データから最新時間断面を抽出することも簡単にできる。

Code 
CREATE OR REPLACE TABLE Temp (x INTEGER, y STRING);
INSERT INTO Temp VALUES 
(1, 'a'), 
(2, 'a'), 
(3, 'b'), 
(-1, 'b');
Code 
SELECT * FROM Temp;

3 b

次の用にQUAFLIFYにつづいてWindow関数を使うことで、Widnow関数の結果に対するフィルタが行える。 これをIdと時間でおこなえばIdごと最新の時間断面データを作成することができる。

Code 
SELECT * 
FROM Temp
QUALIFY ROW_NUMBER() 
  OVER(
    PARTITION BY y
    ORDER BY x DESC
  ) = 1
ORDER BY y
;

3 FILTER

SELECT句でのAGG関数において使えるオプション関数である。 AGG関数に食わせるデータの行に対してWHERE句と同じように 使うことが買おうである。 Window関数ではいまのところ使うことができない。

-- Compare total row count to:
--   The number of rows where i <= 5
--   The number of rows where i is odd
SELECT
    count(*) AS total_rows,
    count(*) FILTER (i <= 5) AS lte_five,
    count(*) FILTER (i % 2 = 1) AS odds
FROM generate_series(1, 10) tbl(i);

-- First generate an example dataset
CREATE TEMP TABLE stacked_data AS
    SELECT
        i,
        CASE WHEN i <= rows * 0.25  THEN 2022
             WHEN i <= rows * 0.5   THEN 2023

5 再帰共通表式

これがあることでSQLで解ける問題の幅が大きく広がる。すごい！！！ ポスグレやMySQLは10年前から実装されていることからミドルウェアが変わってもだいたい実装されているらしい。

実はWITH RECURSIVEを使うことで再帰的な計算をおこなうことができる。 下記は公式ドキュメントにあるようにフィボナッチ数列を計算する。 実装をみると第1行を定義したのちUNION ALLで自分を参照する形になっている。

Code 
WITH RECURSIVE FibonacciNumbers (RecursionDepth, FibonacciNumber, NextNumber) AS (
    -- Base case
    SELECT
        0  AS RecursionDepth,
        0  AS FibonacciNumber,
        1  AS NextNumber
    UNION ALL
    -- Recursive step
    SELECT
        fib.RecursionDepth + 1 AS RecursionDepth,
        fib.NextNumber AS FibonacciNumber,
        fib.FibonacciNumber + fib.NextNumber AS NextNumber
    FROM
        FibonacciNumbers fib
    WHERE
        fib.RecursionDepth + 1 < 24
)
-- Query the CTE
SELECT
    fn.RecursionDepth AS FibonacciNumberIndex,
    fn.FibonacciNumber
FROM
    FibonacciNumbers fn;

Code 
WITH RECURSIVE paths(startNode, endNode, path) AS (
    SELECT -- define the path as the first edge of the traversal
        node1id AS startNode,
        node2id AS endNode,
        [node1id, node2id] AS path
    FROM edge
    WHERE startNode = 1
    UNION ALL
    SELECT -- concatenate new edge to the path
        paths.startNode AS startNode,
        node2id AS endNode,
        array_append(path, node2id) AS path
    FROM paths
    JOIN edge ON paths.endNode = node1id
    -- Prevent adding a repeated node to the path.
    -- This ensures that no cycles occur.
    WHERE node2id != ALL(paths.path)
)
SELECT startNode, endNode, path
FROM paths
ORDER BY length(path), path;

WITH RECURSIVE paths(startNode, endNode, path) AS (
   SELECT -- define the path as the first edge of the traversal
        node1id AS startNode,
        node2id AS endNode,

5.3 2つのノード間の重み付けされていあに最短パスの列挙

Code 
WITH RECURSIVE paths(startNode, endNode, path, endReached) AS (
   SELECT -- define the path as the first edge of the traversal
        node1id AS startNode,
        node2id AS endNode,
        [node1id, node2id] AS path,
        (node2id = 8) AS endReached
     FROM edge
     WHERE startNode = 1
   UNION ALL
   SELECT -- concatenate new edge to the path
        paths.startNode AS startNode,
        node2id AS endNode,
        array_append(path, node2id) AS path,
        max(CASE WHEN node2id = 8 THEN 1 ELSE 0 END)
            OVER (ROWS BETWEEN UNBOUNDED PRECEDING
                           AND UNBOUNDED FOLLOWING) AS endReached
     FROM paths
     JOIN edge ON paths.endNode = node1id
    WHERE NOT EXISTS (SELECT 1
                      FROM paths previous_paths
                      WHERE list_contains(previous_paths.path, node2id))
      AND paths.endReached = 0
)
SELECT startNode, endNode, path
FROM paths
WHERE endNode = 8
ORDER BY length(path), path;

# 再帰共通表式

これがあることでSQLで解ける問題の幅が大きく広がる。すごい！！！
ポスグレやMySQLは10年前から実装されていることからミドルウェアが変わってもだいたい実装されているらしい。

実は`WITH RECURSIVE`を使うことで再帰的な計算をおこなうことができる。
下記は公式ドキュメントにあるようにフィボナッチ数列を計算する。
実装をみると第1行を定義したのち`UNION ALL`で自分を参照する形になっている。

```{sql connection=con}
WITH RECURSIVE FibonacciNumbers (RecursionDepth, FibonacciNumber, NextNumber) AS (
    -- Base case
    SELECT
        0  AS RecursionDepth,
        0  AS FibonacciNumber,
        1  AS NextNumber
    UNION ALL
    -- Recursive step
    SELECT
        fib.RecursionDepth + 1 AS RecursionDepth,
        fib.NextNumber AS FibonacciNumber,
        fib.FibonacciNumber + fib.NextNumber AS NextNumber
    FROM
        FibonacciNumbers fib
    WHERE
        fib.RecursionDepth + 1 < 24
)
-- Query the CTE
SELECT
    fn.RecursionDepth AS FibonacciNumberIndex,
    fn.FibonacciNumber
FROM
    FibonacciNumbers fn;
```

## 2つのノード間の重み付けされていあに最短パスの列挙

```{sql connection=con}
WITH RECURSIVE paths(startNode, endNode, path, endReached) AS (
   SELECT -- define the path as the first edge of the traversal
        node1id AS startNode,
        node2id AS endNode,
        [node1id, node2id] AS path,
        (node2id = 8) AS endReached
     FROM edge
     WHERE startNode = 1
   UNION ALL
   SELECT -- concatenate new edge to the path
        paths.startNode AS startNode,
        node2id AS endNode,
        array_append(path, node2id) AS path,
        max(CASE WHEN node2id = 8 THEN 1 ELSE 0 END)
            OVER (ROWS BETWEEN UNBOUNDED PRECEDING
                           AND UNBOUNDED FOLLOWING) AS endReached
     FROM paths
     JOIN edge ON paths.endNode = node1id
    WHERE NOT EXISTS (SELECT 1
                      FROM paths previous_paths
                      WHERE list_contains(previous_paths.path, node2id))
      AND paths.endReached = 0
)
SELECT startNode, endNode, path
FROM paths
WHERE endNode = 8
ORDER BY length(path), path;
```
