2 QUALIFY

WINDOW()関数の結果を使ったクエリができる。データの読込が１度であるため、相関サブクエリでフィルターしているときにはより効率的、高速になることがある。履歴データから最新時間断面を抽出することも簡単にできる。

Code 
CREATE OR REPLACE TABLE Temp (x INTEGER, y STRING);
INSERT INTO Temp VALUES 
(1, 'a'), 
(2, 'a'), 
(3, 'b'), 
(-1, 'b');
Code 
SELECT * FROM Temp;
4 records
x y
1 a
2 a
3 b
-1 b

次の用にQUAFLIFYにつづいてWindow関数を使うことで、Widnow関数の結果に対するフィルタが行える。 これをIdと時間でおこなえばIdごと最新の時間断面データを作成することができる。

Code 
SELECT * 
FROM Temp
QUALIFY ROW_NUMBER() 
  OVER(
    PARTITION BY y
    ORDER BY x DESC
  ) = 1
ORDER BY y
;
2 records
x y
2 a
3 b

3 FILTER

SELECT句でのAGG関数において使えるオプション関数である。 AGG関数に食わせるデータの行に対してWHERE句と同じように 使うことが買おうである。 Window関数ではいまのところ使うことができない。

Code 
-- Compare total row count to:
--   The number of rows where i <= 5
--   The number of rows where i is odd
SELECT
    count(*) AS total_rows,
    count(*) FILTER (i <= 5) AS lte_five,
    count(*) FILTER (i % 2 = 1) AS odds
FROM generate_series(1, 10) tbl(i);

-- First generate an example dataset
CREATE TEMP TABLE stacked_data AS
    SELECT
        i,
        CASE WHEN i <= rows * 0.25  THEN 2022
             WHEN i <= rows * 0.5   THEN 2023
             WHEN i <= rows * 0.75  THEN 2024
             WHEN i <= rows * 0.875 THEN 2025
             ELSE NULL
             END AS year
    FROM (
        SELECT
            i,
            count(*) OVER () AS rows
        FROM generate_series(1, 100_000_000) tbl(i)
    ) tbl;


-- "Pivot" the data out by year (move each year out to a separate column)
SELECT
    first(i) FILTER (year = 2022) AS "2022",
    first(i) FILTER (year = 2023) AS "2023",
    first(i) FILTER (year = 2024) AS "2024",
    first(i) FILTER (year = 2025) AS "2025",
    first(i) FILTER (year IS NULL) AS "NULLs"

6 縦持ち・横持ち変換

PIVOT、UNPIVOTがあるので、Duckdbには不要であるが、SQLの練習として縦持ち・横持ち変換を書いてみる。

Code 
CREATE OR REPLACE TABLE long (z VARCHAR, x VARCHAR, y INTEGER);
INSERT INTO long VALUES
('s', 'a', 1),
('s', 'b', 2),
('s', 'c', 3);

縦持ちから横持ちへの変換。こちらは集約関数があれば簡単におこなうことができる。

Code 
SELECT 
  z
  ,  SUM(CASE x WHEN 'a' THEN y ELSE NULL END) AS 'a'
  ,  SUM(CASE x WHEN 'b' THEN y ELSE NULL END) AS 'b'
  ,  SUM(CASE x WHEN 'c' THEN y ELSE NULL END) AS 'c'
FROM long
GROUP BY z;
1 records
z a b c
s 1 2 3

7 再帰共通表式

これがあることでSQLで解ける問題の幅が大きく広がる。すごい！！！ ポスグレやMySQLは10年前から実装されていることからミドルウェアが変わってもだいたい実装されているらしい。

実はWITH RECURSIVEを使うことで再帰的な計算をおこなうことができる。 下記は公式ドキュメントにあるようにフィボナッチ数列を計算する。 実装をみると第1行を定義したのちUNION ALLで自分を参照する形になっている。

Code 
WITH RECURSIVE FibonacciNumbers (RecursionDepth, FibonacciNumber, NextNumber) AS (
    -- Base case
    SELECT
        0  AS RecursionDepth,
        0  AS FibonacciNumber,
        1  AS NextNumber
    UNION ALL
    -- Recursive step
    SELECT
        fib.RecursionDepth + 1 AS RecursionDepth,
        fib.NextNumber AS FibonacciNumber,
        fib.FibonacciNumber + fib.NextNumber AS NextNumber
    FROM
        FibonacciNumbers fib
    WHERE
        fib.RecursionDepth + 1 < 24
)
-- Query the CTE
SELECT
    fn.RecursionDepth AS FibonacciNumberIndex,
    fn.FibonacciNumber
FROM
    FibonacciNumbers fn;

7.1 再帰共通表式：グラフ

Code 
CREATE TABLE edge (node1id INTEGER, node2id INTEGER);
INSERT INTO edge
    VALUES
        (1, 3), (1, 5), (2, 4), (2, 5), (2, 10), (3, 1), (3, 5), (3, 8), (3, 10),
        (5, 3), (5, 4), (5, 8), (6, 3), (6, 4), (7, 4), (8, 1), (9, 4);

上記の隣接グラフからあるノードから始まる全てのパスを列挙する。

Code 
WITH RECURSIVE paths(startNode, endNode, path) AS (
    SELECT -- define the path as the first edge of the traversal
        node1id AS startNode,
        node2id AS endNode,
        [node1id, node2id] AS path
    FROM edge
    WHERE startNode = 1
    UNION ALL
    SELECT -- concatenate new edge to the path
        paths.startNode AS startNode,
        node2id AS endNode,
        array_append(path, node2id) AS path
    FROM paths
    JOIN edge ON paths.endNode = node1id
    -- Prevent adding a repeated node to the path.
    -- This ensures that no cycles occur.
    WHERE node2id != ALL(paths.path)
)
SELECT startNode, endNode, path
FROM paths
ORDER BY length(path), path;

7.2 ノードから重み付けされていない最短パスの列挙

Code 
WITH RECURSIVE paths(startNode, endNode, path) AS (
   SELECT -- define the path as the first edge of the traversal
        node1id AS startNode,
        node2id AS endNode,
        [node1id, node2id] AS path
     FROM edge
     WHERE startNode = 1
   UNION ALL
   SELECT -- concatenate new edge to the path
        paths.startNode AS startNode,
        node2id AS endNode,
        array_append(path, node2id) AS path
     FROM paths
     JOIN edge ON paths.endNode = node1id
    -- Prevent adding a node that was visited previously by any path.
    -- This ensures that (1) no cycles occur and (2) only nodes that
    -- were not visited by previous (shorter) paths are added to a path.
    WHERE NOT EXISTS (SELECT 1
                      FROM paths previous_paths
                      WHERE list_contains(previous_paths.path, node2id))
)
SELECT startNode, endNode, path
FROM paths
ORDER BY length(path), path;

7.3 2つのノード間の重み付けされていあに最短パスの列挙

Code 
WITH RECURSIVE paths(startNode, endNode, path, endReached) AS (
   SELECT -- define the path as the first edge of the traversal
        node1id AS startNode,
        node2id AS endNode,
        [node1id, node2id] AS path,
        (node2id = 8) AS endReached
     FROM edge
     WHERE startNode = 1
   UNION ALL
   SELECT -- concatenate new edge to the path
        paths.startNode AS startNode,
        node2id AS endNode,
        array_append(path, node2id) AS path,
        max(CASE WHEN node2id = 8 THEN 1 ELSE 0 END)
            OVER (ROWS BETWEEN UNBOUNDED PRECEDING
                           AND UNBOUNDED FOLLOWING) AS endReached
     FROM paths
     JOIN edge ON paths.endNode = node1id
    WHERE NOT EXISTS (SELECT 1
                      FROM paths previous_paths
                      WHERE list_contains(previous_paths.path, node2id))
      AND paths.endReached = 0
)
SELECT startNode, endNode, path
FROM paths
WHERE endNode = 8
ORDER BY length(path), path;

1 8 1, 5, 8

```{sql connection=con}
CREATE TEMP TABLE stacked_data AS
    SELECT
        i,
        CASE WHEN i <= rows * 0.25  THEN 2022
             WHEN i <= rows * 0.5   THEN 2023
             WHEN i <= rows * 0.75  THEN 2024
             WHEN i <= rows * 0.875 THEN 2025
             ELSE NULL
             END AS year
    FROM (
        SELECT
            i,
            count(*) OVER () AS rows
        FROM generate_series(1, 100_000_000) tbl(i)
    ) tbl;


-- "Pivot" the data out by year (move each year out to a separate column)
SELECT
    first(i) FILTER (year = 2022) AS "2022",
    first(i) FILTER (year = 2023) AS "2023",
    first(i) FILTER (year = 2024) AS "2024",
    first(i) FILTER (year = 2025) AS "2025",
    first(i) FILTER (year IS NULL) AS "NULLs"
FROM stacked_data;
```

# マテリアルズ WITH

デフォルトのWITHは、インラインされるだけなので、同じ処理を重複しておこなう可能性がある。
もし同じテーブルを複数回使うことやWITHの内容が重たい処理の場合であるときには、`Materialized CTEs`を使う方が性能がよくなる。

```{sql connection=con}
WITH iris As MATERIALIZED (
  SELECT * FROM read_csv('./data/iris.csv')
)
SELECT * FROM iris;
```


# 縦持ち・横持ち変換

PIVOT、UNPIVOTがあるので、Duckdbには不要であるが、SQLの練習として縦持ち・横持ち変換を書いてみる。

```{sql connection=con}
CREATE OR REPLACE TABLE long (z VARCHAR, x VARCHAR, y INTEGER);
INSERT INTO long VALUES
('s', 'a', 1),
('s', 'b', 2),
('s', 'c', 3);
```


縦持ちから横持ちへの変換。こちらは集約関数があれば簡単におこなうことができる。

```{sql connection=con}
SELECT 
  z
  ,  SUM(CASE x WHEN 'a' THEN y ELSE NULL END) AS 'a'
  ,  SUM(CASE x WHEN 'b' THEN y ELSE NULL END) AS 'b'
  ,  SUM(CASE x WHEN 'c' THEN y ELSE NULL END) AS 'c'
FROM long
GROUP BY z;
```

横持ちから縦持ちの変換をおこなう。
`SELECT`で切り出した後に、`UNION`でつなぐというのが１つの方法である。

```{sql connection=con}
CREATE OR REPLACE TABLE wide (z VARCHAR, a INTEGER, b INTEGER, c INTEGER);
INSERT INTO wide VALUES
('s', 1, 2, 3),
('t', 4, 5, 6),
('u', 7, 8, 9);

```

```{sql connection=con}
select * from wide;
```


とにかく、展開、つまり行を延ばす必要があるので、ここでは`CROSS JOIN`で行を延ばしてから、
行番号の繰り返しから`CASE`で値を取得している。


```{sql connection=con}
SELECT
  z
  , CASE ii 
      WHEN 0 THEN a 
      WHEN 1 THEN b
      WHEN 2 THEN c
      ELSE NULL END as v
FROM wide
CROSS JOIN range(3) t(ii);
```


シーケンシャルスキャンが１度であることがわかる。

```{sql connection=con}
#| output.var: a
EXPLAIN SELECT
  z
  , CASE ii 
      WHEN 0 THEN a 
      WHEN 1 THEN b
      WHEN 2 THEN c
      ELSE NULL END as v
FROM wide
CROSS JOIN range(3) t(ii);
```

```{r}
a
```



```{sql connection=con}
#| output.var: a
EXPLAIN 
SELECT z, a FROM wide
UNION ALL
SELECT z, b FROM wide
UNION ALL
SELECT z, c FROM wide
```

# 再帰共通表式

これがあることでSQLで解ける問題の幅が大きく広がる。すごい！！！
ポスグレやMySQLは10年前から実装されていることからミドルウェアが変わってもだいたい実装されているらしい。

実は`WITH RECURSIVE`を使うことで再帰的な計算をおこなうことができる。
下記は公式ドキュメントにあるようにフィボナッチ数列を計算する。
実装をみると第1行を定義したのち`UNION ALL`で自分を参照する形になっている。

```{sql connection=con}
WITH RECURSIVE FibonacciNumbers (RecursionDepth, FibonacciNumber, NextNumber) AS (
    -- Base case
    SELECT
        0  AS RecursionDepth,
        0  AS FibonacciNumber,
        1  AS NextNumber
    UNION ALL
    -- Recursive step
    SELECT
        fib.RecursionDepth + 1 AS RecursionDepth,
        fib.NextNumber AS FibonacciNumber,
        fib.FibonacciNumber + fib.NextNumber AS NextNumber
    FROM
        FibonacciNumbers fib
    WHERE
        fib.RecursionDepth + 1 < 24
)
-- Query the CTE
SELECT
    fn.RecursionDepth AS FibonacciNumberIndex,
    fn.FibonacciNumber
FROM
    FibonacciNumbers fn;
```


## 再帰共通表式：グラフ

```{sql connection=con}
CREATE TABLE edge (node1id INTEGER, node2id INTEGER);
INSERT INTO edge
    VALUES
        (1, 3), (1, 5), (2, 4), (2, 5), (2, 10), (3, 1), (3, 5), (3, 8), (3, 10),
        (5, 3), (5, 4), (5, 8), (6, 3), (6, 4), (7, 4), (8, 1), (9, 4);
```

上記の隣接グラフからあるノードから始まる全てのパスを列挙する。

```{sql connection=con}
WITH RECURSIVE paths(startNode, endNode, path) AS (
    SELECT -- define the path as the first edge of the traversal
        node1id AS startNode,
        node2id AS endNode,
        [node1id, node2id] AS path
    FROM edge
    WHERE startNode = 1
    UNION ALL
    SELECT -- concatenate new edge to the path
        paths.startNode AS startNode,
        node2id AS endNode,
        array_append(path, node2id) AS path
    FROM paths
    JOIN edge ON paths.endNode = node1id
    -- Prevent adding a repeated node to the path.
    -- This ensures that no cycles occur.
    WHERE node2id != ALL(paths.path)
)
SELECT startNode, endNode, path
FROM paths
ORDER BY length(path), path;
```

## ノードから重み付けされていない最短パスの列挙

```{sql connection=con}
WITH RECURSIVE paths(startNode, endNode, path) AS (
   SELECT -- define the path as the first edge of the traversal
        node1id AS startNode,
        node2id AS endNode,
        [node1id, node2id] AS path
     FROM edge
     WHERE startNode = 1
   UNION ALL
   SELECT -- concatenate new edge to the path
        paths.startNode AS startNode,
        node2id AS endNode,
        array_append(path, node2id) AS path
     FROM paths
     JOIN edge ON paths.endNode = node1id
    -- Prevent adding a node that was visited previously by any path.
    -- This ensures that (1) no cycles occur and (2) only nodes that
    -- were not visited by previous (shorter) paths are added to a path.
    WHERE NOT EXISTS (SELECT 1
                      FROM paths previous_paths
                      WHERE list_contains(previous_paths.path, node2id))
)
SELECT startNode, endNode, path
FROM paths
ORDER BY length(path), path;
```

## 2つのノード間の重み付けされていあに最短パスの列挙

```{sql connection=con}
WITH RECURSIVE paths(startNode, endNode, path, endReached) AS (
   SELECT -- define the path as the first edge of the traversal
        node1id AS startNode,
        node2id AS endNode,
        [node1id, node2id] AS path,
        (node2id = 8) AS endReached
     FROM edge
     WHERE startNode = 1
   UNION ALL
   SELECT -- concatenate new edge to the path
        paths.startNode AS startNode,
        node2id AS endNode,
        array_append(path, node2id) AS path,
        max(CASE WHEN node2id = 8 THEN 1 ELSE 0 END)
            OVER (ROWS BETWEEN UNBOUNDED PRECEDING
                           AND UNBOUNDED FOLLOWING) AS endReached
     FROM paths
     JOIN edge ON paths.endNode = node1id
    WHERE NOT EXISTS (SELECT 1
                      FROM paths previous_paths
                      WHERE list_contains(previous_paths.path, node2id))
      AND paths.endReached = 0
)
SELECT startNode, endNode, path
FROM paths
WHERE endNode = 8
ORDER BY length(path), path;
```

```{sql connection=con}
SELECT -- define the path as the first edge of the traversal
    node1id AS startNode,
    node2id AS endNode,
    [node1id, node2id] AS path,
    (node2id = 8) AS endReached
FROM edge
WHERE startNode = 1
```
