1 はじめに

1つの車両を用いた多期間の配送計画問題をアツアク。混合整数計画問題(MIP)としてモデリングして 解くことにより配送計画がエラ得ることを示す。

技術的には次の３点が解説される

  1. 巡回セールスマン問題の混合整数計画問題へのモデリング
  2. 数理モデルの表現力を高める補助変数の使い方
  3. 配送パターンの列挙を利用した数理モデルの工夫による求解の効率化

2 課題整理

配送計画では配送車両と地点に関する要件が与えら得る。

  • 配送車両：トラック１台
  • 地点：配送拠点である配送センターが一箇所、配送先であるお店を複数もつ
  • 配送ルート：配送センターを出発し、いくつかのお店を訪問して、配送センターに戻ってくるまでの配送の順番をあらわしたもので、同じお店には２回以上訪問しない

ここでは１ヶ月先までの各お店の発注量がわかるとして、曜日、ドライバーの勤務日数、お店の営業日を考慮したうえで、 角日の配送センターから地点の配送ルートを決めるものである。

  • 配送計画：２０日分の配送ルート、および各配送日にお店に配送する荷物が決定された表

トラックの要件としては１度の最大積載容量がある。

  • 1日に配送できる荷物の重量は4,000kg以下とする

ドライバーに関する要件として労働時間がある。

  • ドライバーの所定労働時間は1日8時間とする
  • ドライバーの所定労働時間を超えて配送業務をおこなう場合、1時間あたり3,000円の残業代が発生する
  • ドライバーの1日の残業時間は3時間以下とする

外部配送業者に関する要件として次がある。

  • 1kgあたり46円とする

最適化の要件として経営者側の視点にたち、「残業費用」と「外注費用」の和を最少化することを目標とする。

3.1.2 決定変数と補助変数

数理モデルに現れる変数のうち、独立した変数として定義するものを決定変数と呼ぶ。 決定変数に従属して値が決まる変数を補助変数と呼ぶ。

ある日\(d\)において地点\(k_1\)から\(k_2\)への移動の有無を表す決定変数を次のように定義する。

\[ x_{d,k_1, k_2}\in \{0, 1\} \]

次に\(d\)における\(k\)に移動する順番を表す補助変数\(u_{d,k}\)を定義する。\(\mathbb{Z}_{\ge 0}\)は 0以上の整数の集合である。

\[ u_{d,k}\in \mathbb{Z}_{\ge 0} \]

次に、配送日において荷物\(r\in R\)を自社配送するかどうかを表す決定変数を定義する.

\[ y_{d,r}\in \{0,1\} \]

最期に配送日の残業時間を表す補助変数を定義する。

\[ h_{d} \in \mathbb{R}_{\ge 0} \]

補助変数は一見すると省略できるようにも見えるが現実の問題を解くことができるモデリングをする場合には必須のテクニックであるう。

実務では非線形な問題を補助変数により線形問題に帰着させることがよくおこなわれる。

3.1.4 サイクルとなる経路

２つ制約式を用いて、「経路がサイクルをつくる」ということを表すことができる。

1つ目はある地点から他の地点への移動の合計は、ある地点への他の地点からの移動の合計と同じになる。

\[ \sum_{k_2\in K}x_{d,k_2, k_1} = \sum_{k_2 \in K}x_{d, k_1, k_2} \]

もう一つは同じ地点に移動するのは1日に一度までという定義である。

\[ \sum_{k_2 \in K}x_{d, k_2, k_1}\le 1 \]

3.1.5 配送センターと通るサイクル

サイクルのうち配送センターを通るものだけに絞るには次の３つの制約をいれる。

まず、ある日に配送センター\(p\)への移動順番は0である。

\[ \u_{d,p}=0 \]

次に値の上限を制約式として与えることで1番目から順番に番号を付けるようにする。

\[ 1\le u_{d,s} \le |K|-1 \]

最期に角配送日においてお店間だけのサイクルを禁止する。\(s_1, s_2 \in S\)であり、荷物の配送先を表す集合。 \(x_{d,s_1, s_2}=1\)のときは\(u_{d, s_1}+1 \le u_{d, s_2}\)となるうえで、 \(s_1\)から\(s_2\)への移動があるので、つまりは\(u_{d, s_1}\le u_{d, s_2}\)という制約が生まれる。 これにより地点間でのサイクルは発生しないことになる。

\[ u_{d, s_1}+1 \le u_{d, s_2}+(|K|-1)(1-x_{d,s_1, s_2}) \]

その他の制約条件は題意から適当に設定することができる。

4 素朴なモデリングの実装と実験

Code 
import pandas as pd
import pulp
from pathlib import Path

data_dir = Path("unshare", "PyOptBook", "5.routing_ver2")

k_df = pd.read_csv(data_dir / "small_locations.csv")
r_df = pd.read_csv(data_dir / "small_orders.csv")
kk_df = pd.read_csv(data_dir / "small_distances.csv")

# 配送日
D = list(range(r_df.b.min(), r_df.e.max() + 1))

# 配送センター
p = k_df[k_df.depo_flag == 1].k.tolist()[0]

# お店のリスト
S = k_df[k_df.depo_flag == 0].k.tolist()

# 地点のリスト
K = [p] + S

# 荷物のリスト
R = r_df.r.tolist()

# 各変数を定義するために必要なリスト
DKK =[(d, k1, k2) for d in D for k1 in K for k2 in K]
DR = [(d, r) for d in D for r in R]
DK = [(d, k) for d in D for k in K]

次に定数などを辞書形式で作成する。

Code 
# 荷物と紐付ける配送先のお店
R2S = {row.r:row.s for row in r_df.itertuples()}

# 荷物に紐付ける重量
R2W = {row.r:row.w for row in r_df.itertuples()}

# 荷物を紐付ける指定配送期間の開始日
R2B = {row.r:row.b for row in r_df.itertuples()}

# 荷物を紐付ける指定配送期間の終了日
R2E = {row.r:row.e for row in r_df.itertuples()}

# 地点間の移動時間
KK2T = {(row.k1, row.k2): row.t for row in kk_df.itertuples()}

素朴なモデリングの実装は次となる。

Code 
prob = pulp.LpProblem("vrp_small3", pulp.LpMinimize)

x = pulp.LpVariable.dicts("x", DKK, cat = "Binary")
u = pulp.LpVariable.dicts("u", DK,  cat = "Integer", lowBound = 0)
y = pulp.LpVariable.dicts("y", DR,  cat = "Binary")
h = pulp.LpVariable.dicts("h", D,   cat = "Continuous", lowBound  = 0)

for d in D:
    for k1 in K:
        prob += pulp.lpSum([x[(d, k1, k2)] for k2 in K]) == pulp.lpSum([x[(d, k2, k1)] for k2 in K])
        prob += pulp.lpSum([x[(d, k2, k1)] for k2 in K]) <= 1


for d in D:
    prob += u[(d, p)] == 0
    for s in S:
        prob += 1 <= u[(d,s)]
        prob += u[(d, s)] <= len(K) - 1
    for s1 in S:
        for s2 in S:
            prob += u[(d, s1)] + 1 <= u[(d, s2)] + (len(K) - 1) * (1 - x[(d, s1, s2)])

for r in R:
    prob += pulp.lpSum([y[(d, r)] for d in D]) <= 1

# 荷物を自社配送するなら配送刃先のお店に訪問
for d in D:
    for r in R:
        tar_s = R2S[r]
        prob += y[(d, r)] <= pulp.lpSum([x[(d, k, tar_s)]for k in K])

# 各配送日について荷物の重量は4,000kg以下
for d in D:
    prob += pulp.lpSum([y[(d, r)] * R2W[r] for r in R]) <= 4000

# 各配送日についてドライバーの残業時間は所定労働時間の8時間を差し引いた労働時間
for d in D:
    prob += pulp.lpSum([KK2T[(k1, k2)] * x[(d, k1, k2)] for k1 in K for k2 in K]) - 8 <= h[d]

for d in D:
    prob += h[d] <= 3

# 荷物は指定配送期間外の配送を禁止
for r in R:
    for d in D:
        if d < R2B[r]:
            prob += y[(d, r)] == 0
        if R2E[r] < d:
            prob += y[(d, r)] == 0

# 配送費用を最少化
zangyo = pulp.lpSum([3000 * h[d] for d in D])
gaityu = pulp.lpSum([46 * R2W[r] * (1 - pulp.lpSum([y[(d, r)] for d in D])) for r in R])
prob += zangyo + gaityu
Code 
solver = pulp.PULP_CBC_CMD()
status = prob.solve(solver)
print('ステータス:', pulp.LpStatus[status])
#> ステータス: Optimal

# 結果の表示
print(f"残業費用： {zangyo.value()}")
#> 残業費用： 10249.22373
print(f"外注費用： {gaityu.value()}")
#> 外注費用： 69000.0
print(f"費用合計： {zangyo.value() + gaityu.value()}")
#> 費用合計： 79249.22373

角日の配送センターから地点の配送ルートを決めるものである。

- 配送計画：２０日分の配送ルート、および各配送日にお店に配送する荷物が決定された表

トラックの要件としては１度の最大積載容量がある。

- 1日に配送できる荷物の重量は4,000kg以下とする

ドライバーに関する要件として労働時間がある。

- ドライバーの所定労働時間は1日8時間とする
- ドライバーの所定労働時間を超えて配送業務をおこなう場合、1時間あたり3,000円の残業代が発生する
- ドライバーの1日の残業時間は3時間以下とする

外部配送業者に関する要件として次がある。

- 1kgあたり46円とする

最適化の要件として経営者側の視点にたち、「残業費用」と「外注費用」の和を最少化することを目標とする。

### 決定変数と補助変数

数理モデルに現れる変数のうち、独立した変数として定義するものを`決定変数`と呼ぶ。
決定変数に従属して値が決まる変数を`補助変数`と呼ぶ。

ある日$d$において地点$k_1$から$k_2$への移動の有無を表す決定変数を次のように定義する。

$$
x_{d,k_1, k_2}\in \{0, 1\}
$$

次に$d$における$k$に移動する順番を表す補助変数$u_{d,k}$を定義する。$\mathbb{Z}_{\ge 0}$は
0以上の整数の集合である。

$$
u_{d,k}\in \mathbb{Z}_{\ge 0}
$$



次に、配送日において荷物$r\in R$を自社配送するかどうかを表す決定変数を定義する.

$$
y_{d,r}\in \{0,1\}
$$

最期に配送日の残業時間を表す補助変数を定義する。

$$
h_{d} \in \mathbb{R}_{\ge 0}
$$


補助変数は一見すると省略できるようにも見えるが現実の問題を解くことができるモデリングをする場合には必須のテクニックであるう。

実務では非線形な問題を補助変数により線形問題に帰着させることがよくおこなわれる。

### サイクルとなる経路


２つ制約式を用いて、「経路がサイクルをつくる」ということを表すことができる。

1つ目はある地点から他の地点への移動の合計は、ある地点への他の地点からの移動の合計と同じになる。

$$
\sum_{k_2\in K}x_{d,k_2, k_1} = \sum_{k_2 \in K}x_{d, k_1, k_2}
$$


もう一つは同じ地点に移動するのは1日に一度までという定義である。

$$
\sum_{k_2 \in K}x_{d, k_2, k_1}\le 1
$$



### 配送センターと通るサイクル

サイクルのうち配送センターを通るものだけに絞るには次の３つの制約をいれる。


まず、ある日に配送センター$p$への移動順番は0である。

$$
\u_{d,p}=0
$$

次に値の上限を制約式として与えることで1番目から順番に番号を付けるようにする。

$$
1\le u_{d,s} \le |K|-1
$$

最期に角配送日においてお店間だけのサイクルを禁止する。$s_1, s_2 \in S$であり、荷物の配送先を表す集合。
$x_{d,s_1, s_2}=1$のときは$u_{d, s_1}+1 \le u_{d, s_2}$となるうえで、
$s_1$から$s_2$への移動があるので、つまりは$u_{d, s_1}\le u_{d, s_2}$という制約が生まれる。
これにより地点間でのサイクルは発生しないことになる。

$$
u_{d, s_1}+1 \le u_{d, s_2}+(|K|-1)(1-x_{d,s_1, s_2})
$$


その他の制約条件は題意から適当に設定することができる。



# 素朴なモデリングの実装と実験

```{python}
import pandas as pd
import pulp
from pathlib import Path

data_dir = Path("unshare", "PyOptBook", "5.routing_ver2")

k_df = pd.read_csv(data_dir / "small_locations.csv")
r_df = pd.read_csv(data_dir / "small_orders.csv")
kk_df = pd.read_csv(data_dir / "small_distances.csv")

# 配送日
D = list(range(r_df.b.min(), r_df.e.max() + 1))

# 配送センター
p = k_df[k_df.depo_flag == 1].k.tolist()[0]

# お店のリスト
S = k_df[k_df.depo_flag == 0].k.tolist()

# 地点のリスト
K = [p] + S

# 荷物のリスト
R = r_df.r.tolist()

# 各変数を定義するために必要なリスト
DKK =[(d, k1, k2) for d in D for k1 in K for k2 in K]
DR = [(d, r) for d in D for r in R]
DK = [(d, k) for d in D for k in K]


```


次に定数などを辞書形式で作成する。

```{python}
# 荷物と紐付ける配送先のお店
R2S = {row.r:row.s for row in r_df.itertuples()}

# 荷物に紐付ける重量
R2W = {row.r:row.w for row in r_df.itertuples()}

# 荷物を紐付ける指定配送期間の開始日
R2B = {row.r:row.b for row in r_df.itertuples()}

# 荷物を紐付ける指定配送期間の終了日
R2E = {row.r:row.e for row in r_df.itertuples()}

# 地点間の移動時間
KK2T = {(row.k1, row.k2): row.t for row in kk_df.itertuples()}


```


素朴なモデリングの実装は次となる。

```{python}
prob = pulp.LpProblem("vrp_small3", pulp.LpMinimize)

x = pulp.LpVariable.dicts("x", DKK, cat = "Binary")
u = pulp.LpVariable.dicts("u", DK,  cat = "Integer", lowBound = 0)
y = pulp.LpVariable.dicts("y", DR,  cat = "Binary")
h = pulp.LpVariable.dicts("h", D,   cat = "Continuous", lowBound  = 0)

for d in D:
    for k1 in K:
        prob += pulp.lpSum([x[(d, k1, k2)] for k2 in K]) == pulp.lpSum([x[(d, k2, k1)] for k2 in K])
        prob += pulp.lpSum([x[(d, k2, k1)] for k2 in K]) <= 1


for d in D:
    prob += u[(d, p)] == 0
    for s in S:
        prob += 1 <= u[(d,s)]
        prob += u[(d, s)] <= len(K) - 1
    for s1 in S:
        for s2 in S:
            prob += u[(d, s1)] + 1 <= u[(d, s2)] + (len(K) - 1) * (1 - x[(d, s1, s2)])

for r in R:
    prob += pulp.lpSum([y[(d, r)] for d in D]) <= 1

# 荷物を自社配送するなら配送刃先のお店に訪問
for d in D:
    for r in R:
        tar_s = R2S[r]
        prob += y[(d, r)] <= pulp.lpSum([x[(d, k, tar_s)]for k in K])

# 各配送日について荷物の重量は4,000kg以下
for d in D:
    prob += pulp.lpSum([y[(d, r)] * R2W[r] for r in R]) <= 4000

# 各配送日についてドライバーの残業時間は所定労働時間の8時間を差し引いた労働時間
for d in D:
    prob += pulp.lpSum([KK2T[(k1, k2)] * x[(d, k1, k2)] for k1 in K for k2 in K]) - 8 <= h[d]

for d in D:
    prob += h[d] <= 3

# 荷物は指定配送期間外の配送を禁止
for r in R:
    for d in D:
        if d < R2B[r]:
            prob += y[(d, r)] == 0
        if R2E[r] < d:
            prob += y[(d, r)] == 0

# 配送費用を最少化
zangyo = pulp.lpSum([3000 * h[d] for d in D])
gaityu = pulp.lpSum([46 * R2W[r] * (1 - pulp.lpSum([y[(d, r)] for d in D])) for r in R])
prob += zangyo + gaityu
```


```{python}
solver = pulp.PULP_CBC_CMD()
status = prob.solve(solver)
print('ステータス:', pulp.LpStatus[status])

# 結果の表示
print(f"残業費用： {zangyo.value()}")
print(f"外注費用： {gaityu.value()}")
print(f"費用合計： {zangyo.value() + gaityu.value()}")
```
