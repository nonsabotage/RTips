1 はじめに

限られた資源のもとでクーポンの最適な配布方法を考える。 顧客は適当な粒度でセグメント化する。

要件整理は次の通り。

  1. 各会員に対してどのパターンのダイレクトメールを送付るのかを決定する
  2. 各会員に対して送付するダイレクトメールはいずれか1パターン１
  3. クーポン付与による来客増加数を最大化する
  4. 会員の予算消費期待値の合計は1000万円以下
  5. 各パターンのダイレクトメールをそれぞれセグメントに属する会員の10%以上に送付する

4 数理モデリング1

Code 
problem = pulp.LpProblem("coupon1", pulp.LpMaximize)

I = cust_df["customer_id"].to_list()
M = [1, 2, 3]

# 決定変数
# 会員iにダイレクトメールmを送付するかどうか
xim = {}
for i in I:
    for m in M:
        xim[(i, m)] = pulp.LpVariable(name=f"xim({i}, {m})", cat = "Binary")

# 制約式
for i in I:
    problem += pulp.lpSum(xim[(i, m)] for m in M) == 1

keys = ["age_cat", "freq_cat"]
cust_prob_df = (
    pd
    .merge(cust_df, prob_df, on = keys)
    .rename(columns = {"prob_dm1":1, "prob_dm2":2, "prob_dm3":3})
)
cust_prob_ver_df = (
    cust_prob_df
    .melt(
        id_vars = ["customer_id"]
        , value_vars = M
        , var_name = "dm"
        , value_name = "prob")
)
# このやり方はスマートだと思う
Pim = cust_prob_ver_df.set_index(["customer_id", "dm"])["prob"].to_dict()

# 目的関数
problem += pulp.lpSum((Pim[(i, m)] - Pim[(i, 1)]) * xim[(i, m)] for i in I for m in [2,3])

Cm = {1:0, 2:1000, 3:2000}
problem += pulp.lpSum(
    Cm[m] * Pim[(i,m)] * xim[(i,m)]
    for i in I
    for m in M
) <= 1_000_000


S  = prob_df["segment_id"].to_list()
Ns = cust_prob_df.groupby("segment_id")["customer_id"].count().to_dict()
Si = cust_prob_df.set_index("customer_id")["segment_id"].to_dict()

for s in S:
    for m in M:
        problem += pulp.lpSum(xim[(i, m)] for i in I if Si[i] == s) >= 0.1 * Ns[s]


# (problem.constraints)

5 数理モデリング2

セグメント送付モデル。

会員のセグメントに対して格パターンのダイレクトメールをどの程度送付するかを決定する

このモデルでは会員をセグメント単位で捉える。

各パターンのダイレクトメールを各セグメントにどの程度の割合で送付するかという問題を決める。

Code 
problem = pulp.LpProblem("coupon2", pulp.LpMaximize)

# 1. 各会員に対してどのパターンのダイレクトメールを送付するかを決定
xsm = {(s, m): pulp.LpVariable(
            name = f"xsm({s}, {m})", 
            cat = "Continuous", 
            lowBound = 0, 
            upBound = 1
        ) 
        for s in S
        for m in M
}

# 2. 各会員に対して送付するダイレクトメールはいずれか１パターン
for s in S:
    problem += pulp.lpSum(xsm[(s, m)] for m in M) == 1

# 3. クーポン付与による来客増加数を最大化する
prob_ver_df = (
    prob_df
    .rename(columns = {"prob_dm1":1, "prob_dm2":2, "prob_dm3":3})
    .melt(id_vars = ["segment_id"], value_vars=M, var_name="dm", value_name="prob")
)
Psm = prob_ver_df.set_index(['segment_id', 'dm'])['prob'].to_dict()

problem += pulp.lpSum(
    Ns[s] * (Psm[(s, m)] - Psm[(s, 1)]) * xsm[(s,m)] 
    for s in S 
    for m in [2, 3]
)

# 4. 会員の予算消費期待値の合計は1000万円以下
Cm = {1:0, 2:1000, 3:2000}
problem += pulp.lpSum(
    Cm[m] * Ns[s] * Psm[(s, m)] * xsm[(s, m)]
    for s in S
    for m in M
) <= 1_000_000

# 5. 各パターンのダイレクトメールをそれぞれセグメントに属する会員の10%以上に送付する
for s in S:
    for m in M:
        problem += xsm[(s, m)] >= 0.1

6 数理モデリング３

送付率の下限値が影響していることが見て取れるので、 送付率下限値最大化モデルを考える。

Code 
problem = pulp.LpProblem(name="coupon3", sense=pulp.LpMaximize)

# 1. 各セグメントへのダイレクトメールの送付率の下限値と各会員に対してどのダイレクトメールを送付するかを決定
xsm = {
    (s, m): pulp.LpVariable(
        name = f"xsm({s}, {m})", 
        cat = "Continuous", 
        lowBound = 0, 
        upBound = 1
    )
    for s in S
    for m in M
}
y = pulp.LpVariable(name="y", cat="Continuous", lowBound=0, upBound=1)

# 2. 各会員に対して送付するダイレクトメールはいずれか1パターン
for s in S:
    problem += pulp.lpSum(xsm[(s, m)] for m in M) == 1

# 3. 各セグメントへのそれぞれのパターンのダイレクトメールの送付率の下限値を最大化
problem += y

# 4. 会員の予算消費期待値の合計は100万円以下
problem += pulp.lpSum(
    Cm[m] * Ns[s] * Psm[(s, m)] * xsm[(s, m)]
    for s in S
    for m in M
) <= 1_000_000

# 5. 各パターンのダイレクトメールを設定した送付率の下限値以上に送付
for s in S:
    for m in M:
        problem += xsm[(s, m)] >= y

5. 各パターンのダイレクトメールをそれぞれセグメントに属する会員の10%以上に送付する

for cost in range(761_850, 3_000_000, 100_000):
    print(f"cost: {cost}")
    problem = pulp.LpProblem(name="DiscountCouponProblem2", sense=pulp.LpMaximize)
    xsm = {
        (s, m): pulp.LpVariable(
            name = f"xsm({s}, {m})", 
            cat = "Continuous", 
            lowBound = 0, 
            upBound = 1
        )
        for s in S
        for m in M
    }
    for s in S:
        problem += pulp.lpSum(xsm[(s, m)] for m in M) == 1
    problem += pulp.lpSum(
        Ns[s] * (Psm[(s, m)] - Psm[(s, 1)]) * xsm[(s, m)] 
        for s in S 
        for m in [2, 3])
    problem += pulp.lpSum(
        Cm[m] * Ns[s] * Psm[(s, m)] * xsm[(s, m)]
        for s in S
        for m in [2, 3]) <= cost
    for s in S:
        for m in M:
            problem += xsm[(s, m)] >= 0.1
    status = problem.solve()
    inc_action = pulp.value(problem.objective)
    cpa = cost / inc_action
    cost_list.append(cost)
    cpa_list.append(cpa)
    inc_action_list.append(inc_action)
    print("- " * 50)

上記の結果に基づいてマーケティングにおける意思決定をおこなうときには、

まとめ

1. 同じ問題を複数の形でモデリングすることができる
2. モデリングの形によって問題の解きやすさが異なる
3. 変数が少なくできる場合には少なくした方がよい
4. 連続変数と離散変数がどちらも利用できる場合には連続変数を利用したほうがよい場合が多い
5. 費用対効果や公平性の観点で結果を確認してモデルをブラッシュアップ
6. 施策のパラメータを変化させながら結果を評価すること

特に６が行えるのが数理モデリングをおこなうよいところである。
