1 LightGBM分類

同じデータをLightGBMを使う分類する. 勾配ブースティング分類では、勾配ブースティング回帰の予測値をシグモイド関数に入力して「ラベル１」の確率で、ラベルを予測する。

\[ p = \frac{1}{1 + \text{exp}(-(\hat{y}^{(0)}+\sum_{k=1}^K w_k(x)))} \]

よってロジット値は次とな。

\[ \text{logit}(p(y=1\|X))=\log\frac{p(y=1\|X)}{1-p(y=1\|X)}=\hat{y}^{(0)}+\sum_{k=1}^K w_k(x) \]

Code 
X = df.drop(labels = ["income"], axis = 1)
y = df["income"]
Code 
X_train, X_test, y_train, y_test = train_test_split(
    X, y, 
    test_size=.2, 
    shuffle=True, 
    stratify=y, 
    random_state=0
)

label encodingを使いカテゴリ変数の数値に変換する。決定木系のアルゴリズムではlabel encodingを使うとのことである。

Label Encodingはカテゴリカルな変数に対して数値を割り当てる処理である。順序関係がカテゴリー間に設定されてしまうことになる。

ChatGPTによると決定木系だからといって、label encoding為てまうのはよくなさそうである。 順番が欲しいならmapなどで置換することが良さそうである。

ここではとりあえず本の通りに処理を進めてみる。

木を可視化してみるとカテゴリ変数を使った「0||5」という条件がでてきた。これは０か５のときということなので、どうやらlightGBMのときには素直にラベルエンコーディングしても大丈夫のようである。

なお、LightGBMは自動で数値に変換する機能がついているが、後後に使うSHAP値は数値しか受け付けないためこの処理が必要である。
