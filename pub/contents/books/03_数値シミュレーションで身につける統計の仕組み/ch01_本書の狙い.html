1 シミュレーションでわかること

「相関係数が0.5のときの２変量確率変数をN個観測したときに実際に得られる相関係数」 についてシミュレーションをおこなう。これを観ると、サンプルサイズが２５では、非常にバラツキが大きいことがわかる。

Code 
rho <- .5
n <- 25
iter <- 1000

r <- rep(0, iter)

set.seed(123)
for (i in 1:iter) {
  Y1 <- rnorm(n, 0, 1)
  Y2 <- Y1 * rho + rnorm(n, 0, (1 - rho^2)^.5)
  r[i] <- cor(Y1, Y2)
}


hist(r)

サンプルサイズを増やすと推定値のバラツキが小さくなることがわかる。

Code 
rho <- .5
n <- 100
iter <- 1000

r <- rep(0, iter)

set.seed(123)
for (i in 1:iter) {
  Y1 <- rnorm(n, 0, 1)
  Y2 <- Y1 * rho + rnorm(n, 0, (1 - rho^2)^.5)
  r[i] <- cor(Y1, Y2)
}


hist(r)

さらに検定もおこなう. ここでは無相関検定\(\rho = 0\)をおこなう。 具体的には無相関検定をおこなってp値を求める。求めたp値が、0.05を下回る 回数の割合を求める.

この結果を見ると仮定が正しいときに、p値が0.05を下回る確率は5%であることがわかる。

Code 
rho <- .0
n <- 25
iter <- 10000

r <- rep(0, iter)

set.seed(123)
for (i in 1:iter) {
  Y1 <- rnorm(n, 0, 1)
  Y2 <- Y1 * rho + rnorm(n, 0, (1 - rho^2)^.5)
  r[i] <- cor.test(Y1, Y2)$p.value
}

ifelse(r <= .05, 1, 0) |> mean()
#> [1] 0.0488

検定が適切におこなわれれば、p値は一様分布にしたがう。

Code 
hist(r)

また、正しく検定がおこわれればp値はサンプルサイズに依らない。

Code 
rho <- .0
n <- 2500
iter <- 10000

r <- rep(0, iter)

set.seed(123)
for (i in 1:iter) {
  Y1 <- rnorm(n, 0, 1)
  Y2 <- Y1 * rho + rnorm(n, 0, (1 - rho^2)^.5)
  r[i] <- cor.test(Y1, Y2)$p.value
}

ifelse(r <= .05, 1, 0) |> mean()
#> [1] 0.0486

誤った尾統計分析としてあるのは、検定において データの検定結果を見てから「も少しで有意になるからサンプルサイズをもう少し大きくしよう」とデータを足すことです。

これが問題となることをシミュレーションで確認する。このシミュレーションから、後からデータを追加することは、p値のバイアスを生むことがわかる。

Code 
## 設定と準備
rho <- 0
n <- 25
iter <- 10000
alpha <- 0.05

# 結果を格納するオブジェクト
p <- rep(0, iter)

## シミュレーション
set.seed(123)
for (i in 1:iter) {
  # 最初のデータ
  Y1 <- rnorm(n, 0, 1)
  Y2 <- Y1 * rho + rnorm(n, 0, (1 - rho^2)^0.5)
  p[i] <- cor.test(Y1, Y2)$p.value
  # データ追加
  count <- 0
  ## p値が5％を下回るか、データが当初の倍になるまで増やし続ける
  while (p[i] >= alpha && count < n * 2) {
    # 有意ではなかったとき、それぞれの変数に1つデータを追加
    Y1_add <- rnorm(1, 0, 1)
    Y1 <- c(Y1, Y1_add)
    Y2 <- c(Y2, Y1_add * rho + rnorm(1, 0, (1 - rho^2)^0.5))
    p[i] <- cor.test(Y1, Y2)$p.value
    count <- count + 1
  }
}

ifelse(p < .05, 1, 0) |> mean()
#> [1] 0.1791
