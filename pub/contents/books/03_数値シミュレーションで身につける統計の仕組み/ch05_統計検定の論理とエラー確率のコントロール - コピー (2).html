1 統計的検定の論理

検定はあらかじめ\(\alpha\)と\(\beta\)を設定しておき、 その確率にタイプⅠ、タイプⅡエラー 確率が抑えられるように手続きを設計することが重要である。

重要なのは低い\(\alpha\)と\(\beta\)を設定することではなく、 推論のエラー確率が定めた\(\alpha\)と\(\beta\)を下回るように、 手続きを設定することである。

2 エラー確率のシミュレーション

  1. 帰無仮説が正しい状況からデータ生成
  2. そのデータから帰無仮説が棄却される確率を傾斜すうｒ

とうステップで、エラー確率のシミュレーションをおこなう。

Code 
n <- c(20, 20) # サンプルサイズ
mu <- c(0, 0) # 母平均が等しい設定にする
sigma <- 2 # 母標準偏差
iter <- 10000 # シミュレーション回数
alpha <- 0.05 # 有意水準
# 結果を格納するオブジェクト
pvalue <- rep(0, iter) # p値を格納する
# シミュレーション
set.seed(123)
for (i in 1:iter) {
  Y1 <- rnorm(n[1], mu[1], sigma) # 群1のデータ生成
  Y2 <- rnorm(n[2], mu[2], sigma) # 群2のデータ生成
  result <- t.test(Y1, Y2, var.equal = TRUE) # t検定の結果を出力
  pvalue[i] <- result$p.value # p値を取得
}
## 結果
# 誤って帰無仮説を棄却した確率を計算
type1error <- ifelse(pvalue < alpha, 1, 0) |> mean()
# 出力
type1error
#> [1] 0.0492

任意の確率分布に従う値を確率に変換した値は、 一様分布に従う

Code 
rnorm(10000, 0, 1) |>
pnorm(mean = 0, sd = 1) |>
hist()

これは次の式からわかる。\(t\)が検定統計量であるとする。

\[ \begin{align} p &= F(t)\\ t &= F^{-1}(p)\\ g(p) &= f(t=F^{-1}(p)) \left| \frac{dt}{dp} \right|\\ \frac{dt}{dp} &= \frac{1}{f(t=F^{-1}(p))}\\ g(p) &= 1 \end{align} \]

これはP値でも同じである。 実際には\(p = 1 - F(t)\)なのでここから\(p\)値の確率密度を 求めると一様分布であることがわかる。

2.1 \(t\)検定の等分散の仮定からの逸脱

等分散が満たされないときに、 エラー確率が制御できないことを確認する。

Code

sigma2 <- c(0.2, 0.5, 0.75, 1, 1.5, 2, 5) # 母標準偏差のパターン

alpha <- 0.05 # 有意水準

    Y2 <- rnorm(n[2], mu[2], sigma2[i])

3 一元配置分散分析

3群以上の平均値の差を同時に検定する手法である。

複数の検定が同時に棄却される確率は それぞれがエラー確率を制御していたとしても 大きくなる。

いま、すべて平均値が同じ３つの郡があり、 そこからペアを取り出して、それぞれt検定を行うことにする。 さいごに、少なくとも１つの検定が間違えている確率を計算する。

Code 

alpah <- .05
k <- 3
n     <- c(20, 20, 20)
mu    <- c(5, 5, 5)
sigma <- c(2 ,2, 2)
iter <- 1000

pvalue12 <- rep(0, iter)
pvalue13 <- rep(0, iter)
pvalue23 <- rep(0, iter)


set.seed(1234)
for (i in 1:iter) {
  Y1 <- rnorm(n[1], mu[1], sigma[1])
  Y2 <- rnorm(n[2], mu[2], sigma[2])
  Y3 <- rnorm(n[3], mu[3], sigma[3])
  
  result12 <- t.test(Y1, Y2)$p.value
  result13 <- t.test(Y1, Y3)$p.value
  result23 <- t.test(Y2, Y3)$p.value

  pvalue12[i] <- result12
  pvalue13[i] <- result13
  pvalue23[i] <- result23
}


type1error <- 
  ifelse(
   pvalue12 < alpha | pvalue13 < alpha | pvalue23 < alpha, 
    1, 
    0
  ) |>
  mean()

type1error
#> [1] 0.128

これは３つの帰無仮説がすべてコントロールできる確率は\((1-\alpha)^3\)になるためである。これは検定の多重性の問題と飛ばれる。

検定の多重性は分散分析により部分的に解決される。

一元配置分散分析は、複数の郡の平均値の差を検定するもんである。 ３群の比較の際には\(\mu_1 = \mu_2 = \mu_3\)が帰無仮説となる。

一方で対立仮説は、いずれかの群の平均値が異なるということである。 つまり棄却されたときにどの群と群との間だに、 差があるのかを主張することまではできない。

Code 
# 設定と準備
k <- 3
n <- rep(20, k)
mu <- c(6, 5, 4)
sigma <- rep(2, k)

# 各軍について、1, 2, 3, という数列を20ずつ作る
x <- c(
  rep(1, n[1]),
  rep(2, n[2]),
  rep(3, n[3])
) |> as.factor()

x

#> Levels: 1 2 3

# 一元配置分散分析

3群以上の平均値の差を同時に検定する手法である。

複数の検定が同時に棄却される確率は
それぞれがエラー確率を制御していたとしても
大きくなる。

いま、すべて平均値が同じ３つの郡があり、
そこからペアを取り出して、それぞれt検定を行うことにする。
さいごに、少なくとも１つの検定が間違えている確率を計算する。


```{r}
alpah <- .05
k <- 3
n     <- c(20, 20, 20)
mu    <- c(5, 5, 5)
sigma <- c(2 ,2, 2)
iter <- 1000

pvalue12 <- rep(0, iter)
pvalue13 <- rep(0, iter)
pvalue23 <- rep(0, iter)


set.seed(1234)
for (i in 1:iter) {
  Y1 <- rnorm(n[1], mu[1], sigma[1])
  Y2 <- rnorm(n[2], mu[2], sigma[2])
  Y3 <- rnorm(n[3], mu[3], sigma[3])
  
  result12 <- t.test(Y1, Y2)$p.value
  result13 <- t.test(Y1, Y3)$p.value
  result23 <- t.test(Y2, Y3)$p.value

  pvalue12[i] <- result12
  pvalue13[i] <- result13
  pvalue23[i] <- result23
}


type1error <- 
  ifelse(
   pvalue12 < alpha | pvalue13 < alpha | pvalue23 < alpha, 
    1, 
    0
  ) |>
  mean()

type1error


```

これは３つの帰無仮説がすべてコントロールできる確率は$(1-\alpha)^3$になるためである。これは検定の多重性の問題と飛ばれる。

検定の多重性は分散分析により部分的に解決される。

一元配置分散分析は、複数の郡の平均値の差を検定するもんである。
３群の比較の際には$\mu_1 = \mu_2 = \mu_3$が帰無仮説となる。

一方で対立仮説は、いずれかの群の平均値が異なるということである。
つまり棄却されたときにどの群と群との間だに、
差があるのかを主張することまではできない。
