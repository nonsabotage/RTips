1 はじめに

patchworkはggplot2の拡張パッケージである。wrap_plotは非常に強力に、ggplot2のプロットを組み合わせることができる。

HPをみると一目ですごさがわかる。 軸を共有することも可能であるし、別々の軸を持つこともできる。

2 Basic Example

Code 
library(ggplot2)
library(patchwork)

p1 <- ggplot(mtcars) + geom_point(aes(mpg, disp))
p2 <- ggplot(mtcars) + geom_boxplot(aes(gear, disp, group = gear))

p1 + p2

上下に組み合わせることも可能である。

Code 
p3 <- ggplot(mtcars) + geom_smooth(aes(disp, qsec))
p4 <- ggplot(mtcars) + geom_bar(aes(carb))

(p1 | p2 | p3) /
      p4
#> `geom_smooth()` using method = 'loess' and formula = 'y ~ x'

3 Getting Started

Code 
library(ggplot2)
p1 <- ggplot(mtcars) + 
  geom_point(aes(mpg, disp)) + 
  ggtitle('Plot 1')

p2 <- ggplot(mtcars) + 
  geom_boxplot(aes(gear, disp, group = gear)) + 
  ggtitle('Plot 2')

p3 <- ggplot(mtcars) + 
  geom_point(aes(hp, wt, colour = mpg)) + 
  ggtitle('Plot 3')

p4 <- ggplot(mtcars) + 
  geom_bar(aes(gear)) + 
  facet_wrap(~cyl) + 
  ggtitle('Plot 4')

単純に凡例を付けると、それはグラフをくっつけた新しいオブジェクトに対するものではなく、 通常のggplotのプロットに対するものであることに注意する。

Code 
p1 + p2 + labs(subtitle = 'This will appear in the last plot')

3.1 Controlling layout

通常は自動で、カラム数が２のレイアウトになる。

Code 
p1 + p2 + p3 + p4
Code 
p1 + p2 + p3 + p4 + plot_layout(nrow = 3, byrow = FALSE)

3.2 Stacking and packing plots

1行や1列のレイアウトを指定することにより、グラフの大きさを他のグラフと変更することも可能である。 もしくはレイアウトのショートカットとして|や/を使うこともできる。

/の方が先に演算されているように見える。

Code 
p1 | (p2 | p3) / p4
Code 
p1 | p2 | p3 / p4

3.3 Annotating the composition

plot_annotationを使うことで、結合したプロット全体に対するアノテーションを与えることが可能である

Code 
(p1 | (p2 / p3)) + 
  plot_annotation(title = "The surprising story about mtcars")

プロットそれぞれにタグを与えることが可能である。 これによりサブプロットの見分けが付きやすくなる。

Code 
p1 + p2 + p3 + plot_annotation(tag_levels = "I")

3.4 Controlling guides

plot_layoutのguides引数を使うことで、凡例の制御が可能である。 そのままプロットするとサブプロットごとの位置に凡例が表示されるが、guidesを使うことで、 プロット全体に対することがわかりやすい位置に配置される。

Code 
p1 + p2 + p3 + p4
Code 
p1 + p2 + p3 + p4 + plot_layout(guides = "collect")

初期設定の設定はautoである。keepにすると、かならずサブプロットのプロット領域に凡例が配置されることになる。

次は複数の凡例を有する場合のプロットである。

Code 
p1a <- ggplot(mtcars) + geom_point(aes(mpg, disp, colour = mpg, size = wt)) + 
  ggtitle("PLot 1a")

p1a | (p2 / p3)

上記の状態でguidesをcollectにすると、凡例がプロット全体に配置される。

Code 
(p1a | (p2 / p3)) + plot_layout(guides = "collect")

凡例を凡例だけの場所を準備してそこにプロットすることも可能である。

Code 
p1 + p2 + p3 + guide_area() + plot_layout(guides = "collect")

3.5 Controlling scales

plot_layoutのaxes=collect引数を使うことで、スケールの制御が可能である。

通常の+では軸は同期された表現にならないが、上記の設定により同期された表現となる。

Code 
p1 + p2 + plot_layout(axes = "collect")

同期はするが、ラベルを消さないことも可能である。

Code 
p1 + (p2 + scale_y_continuous(position = "right")) + plot_layout(axis_titles = "collect")

3.6 Adding empty area

Code 
p1 + plot_spacer() + p2 + plot_spacer() + p3 + plot_spacer()
Code 
(p1 + plot_spacer() + p2) / (plot_spacer() + p3 + plot_spacer())

3.7 Controlling the grid

Code 
p1 + p2 + p3 + p4 + 
  plot_layout(widths = c(2, 1), byrow = FALSE, ncol = 2)

3.8 Moving beyond the grid

より柔軟なレイアウトの指定方法である

Code 
layout <- "
##BBBB
AACCDD
##CCDD
"
p1 + p2 + p3 + p4 + 
  plot_layout(design = layout)

areaconstructorを使うことでさらにプログラミックなものを作成することが可能である。

Code 
layout <- c(
  area(t = 2, l = 1, b = 5, r = 4),
  area(t = 1, l = 3, b = 3, r = 5), 
  area(t = 3, l = 2, b = 4, r = 2.5)
)
p1 + p2 + p3 + 
  plot_layout(design = layout)

3.9 Fixed aspect plots

固定軸でのプロットアセンブリは特別なケースである。coord_fixed, coord_polar, coord_sf。

デフォルトな固定軸に合わせられる。

Code 
p_fixed <- ggplot(mtcars) + 
  geom_point(aes(hp, disp)) + 
  ggtitle('Plot F') + 
  coord_fixed()
p_fixed + p1 + p2 + p3

合わせたくないときには、幅を指定すればよい。アスペクト比が維持されていることがわかるｓｓ

Code 
p_fixed + p1 + p2 + p3 + plot_layout(widths = 1)

3.10 Avoiding alignment

たとえば、captionが長いがそれに合わせて他のグラフについて、余白を調整したくないときがある。 そのときには次のような処理をおこなう。

Code 
p2mod <- p2 + labs(x = "This is such a long\nand important label that\nit has to span many lines")
p1 | p2mod
Code 
free(p1) | p2mod
Code 
p1 + inset_element(p2, left = 0.6, bottom = 0.6, right = 1, top = 1)

3.11 Insets

Code 
p1 + inset_element(p2, left = 0.6, bottom = 0.6, right = 1, top = 1)

npcという変数を使うことで、エリアの座標を0, 1にした相対値で指定することが可能となる

Code 

p1 + inset_element(
  p2, 
  left = 0.5, 
  bottom = 0.5, 
  right = unit(1, 'npc') - unit(1, 'cm'), 
  top = unit(1, 'npc') - unit(1, 'cm')
)

# はじめに

patchworkはggplot2の拡張パッケージである。`wrap_plot`は非常に強力に、ggplot2のプロットを組み合わせることができる。

[HP](https://patchwork.data-imaginist.com/)をみると一目ですごさがわかる。
軸を共有することも可能であるし、別々の軸を持つこともできる。

## Controlling guides

`plot_layout`の`guides`引数を使うことで、凡例の制御が可能である。
そのままプロットするとサブプロットごとの位置に凡例が表示されるが、`guides`を使うことで、


```{r}
p1 + p2 + p3 + p4 + plot_layout(guides = "collect")
```
