tidyverseはメタパッケージです。

2.1 dplyr 1.1.0: pick, reframe, and arrange

2.1.1 pick

Code 
df <- tibble(
  x_1 = c(1, 3, 2, 1, 2), 
  x_2 = 6:10, 
  w_4 = 11:15, 
  y_2 = c(5, 2, 4, 0, 6)
)

df |>
  summarise(
    n_x = ncol(across(starts_with("x"))),
    n_y = ncol(across(starts_with("y")))
  )
#> # A tibble: 1 × 2
#>     n_x   n_y
#>   <int> <int>
#> 1     2     1

acrossは関数の適用対象となカラムに関心があるので名前が少し不自然である。 pickはカラムを選択していることが自然にわかるものである。

Code 
df |> 
  summarise(
    n_x = ncol(pick(starts_with("x"))), 
    n_y = ncol(pick(starts_with("y")))
  )
#> # A tibble: 1 × 2
#>     n_x   n_y
#>   <int> <int>
#> 1     2     1

pickはたとえばdense_rankで複数のカラムを使うときに有効である。

Code 
df |> 
  mutate(
    rank1 = dense_rank(x_1), 
    rank2 = dense_rank(pick(x_1, y_2))
  )
#> # A tibble: 5 × 6
#>     x_1   x_2   w_4   y_2 rank1 rank2
#>   <dbl> <int> <int> <dbl> <int> <int>
#> 1     1     6    11     5     1     2
#> 2     3     7    12     2     3     5
#> 3     2     8    13     4     2     3
#> 4     1     9    14     0     1     1
#> 5     2    10    15     6     2     4

.fnsを使わないacrossを廃止することはまだないが、今後はpickが代替手段となる.

2.1.2 reframe

集約関数を適用したときに複数行を返すことができるのは有用であると認識されているが、 summariseでおこなうのは適当でないと考えられる。

Code 
df <- tibble(
  g = c(1, 1, 1, 2, 2),
  x = c(4, 3, 6, 2, 8),
  y = c(5, 1, 2, 8, 9)
)

df
#> # A tibble: 5 × 3
#>       g     x     y
#>   <dbl> <dbl> <dbl>
#> 1     1     4     5
#> 2     1     3     1
#> 3     1     6     2
#> 4     2     2     8
#> 5     2     8     9

sd(x)とするところが、xとしてしまって値がリサイクルされている。 この場合、summariseで集約していないので、警告が発生する。

Code 
df |>
  summarise(
    x_average = mean(x),
    x_sd = x, # Oops
    .by = g
  )
#> Warning: Returning more (or less) than 1 row per `summarise()` group was deprecated in
#> dplyr 1.1.0.
#> ℹ Please use `reframe()` instead.
#> ℹ When switching from `summarise()` to `reframe()`, remember that `reframe()`
#>   always returns an ungrouped data frame and adjust accordingly.
#> # A tibble: 5 × 3
#>       g x_average  x_sd
#>   <dbl>     <dbl> <dbl>
#> 1     1      4.33     4
#> 2     1      4.33     3
#> 3     1      4.33     6
#> 4     2      5        2
#> 5     2      5        8
#> Warning: Returning more (or less) than 1 row per `summarise()` group was deprecated in
#> dplyr 1.1.0.
#> ℹ Please use `reframe()` instead.
#> ℹ When switching from `summarise()` to `reframe()`, remember that `reframe()`
#>   always returns an ungrouped data frame and adjust accordingly.
#> # A tibble: 5 × 3
#>       g x_average  x_sd
#>   <dbl>     <dbl> <dbl>
#> 1     1      4.33     4
#> 2     1      4.33     3
#> 3     1      4.33     6
#> 4     2      5        2
#> 5     2      5        8

そこで、reframeを使う。また、enframe、deframeも使う。

  • enframe: Takes a vector, returns a data frame
  • deframe: Takes a data frame, returns a vector
  • reframe: Takes a data frame, returns a data frame

次の例では、グループ化した状態で、カラムxを引数としてquantile_dfが実行されている。 group_by -> map -> bind_rowsの流れを非常に簡潔に記述できていることがわかる。

Code 
quantile_df <- function(x, probs = c(0.25, 0.5, 0.75)) {
  tibble(
    value = quantile(x, probs, na.rm = TRUE),
    prob = probs
  )
}
df |>
  reframe(quantile_df(x), .by = g)
#> # A tibble: 6 × 3
#>       g value  prob
#>   <dbl> <dbl> <dbl>
#> 1     1   3.5  0.25
#> 2     1   4    0.5 
#> 3     1   5    0.75
#> 4     2   3.5  0.25
#> 5     2   5    0.5 
#> 6     2   6.5  0.75
#> # A tibble: 6 × 3
#>       g value  prob
#>   <dbl> <dbl> <dbl>
#> 1     1   3.5  0.25
#> 2     1   4    0.5 
#> 3     1   5    0.75
#> 4     2   3.5  0.25
#> 5     2   5    0.5 
#> 6     2   6.5  0.75

acrossと組み合わせることで処理を拡張、つまり複数の列に対して処理がおこなえるようになる。

これはつまり、横持ち状態のデータフレームに対しても処理が簡潔におこなえるということになる。

Code 
df |>
  reframe(
    across(x:y, quantile_df), 
    .by = g
  )
#> # A tibble: 6 × 3
#>       g x$value $prob y$value $prob
#>   <dbl>   <dbl> <dbl>   <dbl> <dbl>
#> 1     1     3.5  0.25    1.5   0.25
#> 2     1     4    0.5     2     0.5 
#> 3     1     5    0.75    3.5   0.75
#> 4     2     3.5  0.25    8.25  0.25
#> 5     2     5    0.5     8.5   0.5 
#> 6     2     6.5  0.75    8.75  0.75

データをunpackすることも可能である。

Code 
df |> 
  reframe(
    across(x:y, quantile_df, .unpack = TRUE), 
    .by = g
  )
#> # A tibble: 6 × 5
#>       g x_value x_prob y_value y_prob
#>   <dbl>   <dbl>  <dbl>   <dbl>  <dbl>
#> 1     1     3.5   0.25    1.5    0.25
#> 2     1     4     0.5     2      0.5 
#> 3     1     5     0.75    3.5    0.75
#> 4     2     3.5   0.25    8.25   0.25
#> 5     2     5     0.5     8.5    0.5 
#> 6     2     6.5   0.75    8.75   0.75

enframeについても試してみる。 stack, unstackと同じ処理のようにみえる。

Code 
enframe(1:3)
#> # A tibble: 3 × 2
#>    name value
#>   <int> <int>
#> 1     1     1
#> 2     2     2
#> 3     3     3
enframe(c(a = 5, b = 7))
#> # A tibble: 2 × 2
#>   name  value
#>   <chr> <dbl>
#> 1 a         5
#> 2 b         7
enframe(list(one = 1, two = 2:3, three = 4:6))
#> # A tibble: 3 × 2
#>   name  value    
#>   <chr> <list>   
#> 1 one   <dbl [1]>
#> 2 two   <int [2]>
#> 3 three <int [3]>
deframe(enframe(3:1))
#> 1 2 3 
#> 3 2 1
deframe(tibble(a = 1:3))
#> [1] 1 2 3
deframe(tibble(a = as.list(1:3)))
#> [[1]]
#> [1] 1
#> 
#> [[2]]
#> [1] 2
#> 
#> [[3]]
#> [1] 3

## dplyr 1.1.0: pick, reframe, and arrange

### pick

```{r}
df <- tibble(
  x_1 = c(1, 3, 2, 1, 2), 
  x_2 = 6:10, 
  w_4 = 11:15, 
  y_2 = c(5, 2, 4, 0, 6)
)

df |>
  summarise(
    n_x = ncol(across(starts_with("x"))),
    n_y = ncol(across(starts_with("y")))
  )

```

`across`は関数の適用対象となカラムに関心があるので名前が少し不自然である。
`pick`はカラムを選択していることが自然にわかるものである。

```{r}
df |> 
  summarise(
    n_x = ncol(pick(starts_with("x"))), 
    n_y = ncol(pick(starts_with("y")))
  )
```

`pick`はたとえば`dense_rank`で複数のカラムを使うときに有効である。

```{r}
df |> 
  mutate(
    rank1 = dense_rank(x_1), 
    rank2 = dense_rank(pick(x_1, y_2))
  )
```

`.fns`を使わない`across`を廃止することはまだないが、今後は`pick`が代替手段となる.

### reframe

集約関数を適用したときに複数行を返すことができるのは有用であると認識されているが、
`summarise`でおこなうのは適当でないと考えられる。


```{r}
df <- tibble(
  g = c(1, 1, 1, 2, 2),
  x = c(4, 3, 6, 2, 8),
  y = c(5, 1, 2, 8, 9)
)

df
```

enframeについても試してみる。
`stack`, `unstack`と同じ処理のようにみえる。

```{r}
deframe(enframe(3:1))
deframe(tibble(a = 1:3))

```
