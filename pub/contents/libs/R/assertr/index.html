1 はじめに

エンジンのシリンダー数ごとにガロンあたりの平均走行距離を求めたいとします。まず、列 “mpg”、“vs”、および “am” があること、データセットに 10 個以上の観測値が含まれていること、‘miles per gallon’ (mpg) の列が正の数であること、‘miles per gallon’ (mpg) の列に平均から 4 標準偏差の外側のデータムが含まれていないことを確認します。 - AM列とVS列(それぞれ自動/手動とV/ストレートエンジン)に0と1のみが含まれていること - 各行には最大2つのNAが含まれます - 各行は、“mpg”、“am”、および “wt” 列の間で一意です。 - 各行のマハラノビス距離は、すべての距離の絶対偏差の中央値の10以内です(外れ値検出の場合)

2.3 verify vs. assert

式を引数にとるか、述語関数やカラムを引数にとるかが異なる。 またverifyは引数全体に対する評価であり、全体を評価してから違反がないかを考えている。 このため行単位での問題の特定が出来ない。

2.4 asserts predicates

述語関数には次のようなものがある。

  • not_naはNAじゃないことを確認する
  • within_boundsは数値が与えられた範囲に含まれているかを確認する
  • in_setは集合のメンバーであるかどうかを確認する
  • is_uniqはユニークかどうかを確認する

2.10 combining chains of assertions

パイプラインだと1つの処理が失敗するとそこから先の処理は実施されない。 一方で、複数のチェックをリポートしてもらいたいときがある。

Code 
mtcars %>%
  chain_start %>%
  verify(nrow(mtcars) > 10) %>%
  verify(mpg > 0) %>%
  insist(within_n_sds(4), mpg) %>%
  assert(in_set(0,1), am, vs) %>%
  chain_end %>%
  group_by(cyl) %>%
  summarise(avg.mpg=mean(mpg))
#> # A tibble: 3 × 2
#>     cyl avg.mpg
#>   <dbl>   <dbl>
#> 1     4    26.7
#> 2     6    19.7
#> 3     8    15.1

2.11 advanced: creating your own predicate generators for insist

Code 
within_3_iqrs <- function(a_vector){
  the_median <- median(a_vector)
  the_iqr <- IQR(a_vector)
  within_bounds((the_median-the_iqr*3), (the_median+the_iqr*3))
}
mtcars %>%
  insist(within_3_iqrs, mpg) %>%
  group_by(cyl) %>%
  summarise(avg.mpg=mean(mpg))
#> # A tibble: 3 × 2
#>     cyl avg.mpg
#>   <dbl>   <dbl>

# はじめに

[assertr](https://docs.ropensci.org/assertr/)はアサーションのためのライブラリである。
特にデータフレーム操作に対して特化されており、パイプラインと連携して利用することができる。
データフレームの品質をチェックする、つまりPythonのpanderaのような使い方に適している、とのことである。

関数には次がある。

```{r}
#| output: asis

  cat(sep = "\n")
```

たとえば次のような演算を考える。

> エンジンのシリンダー数ごとにガロンあたりの平均走行距離を求めたいとします。まず、列 "mpg"、"vs"、および "am" があること、データセットに 10 個以上の観測値が含まれていること、'miles per gallon' (mpg) の列が正の数であること、'miles per gallon' (mpg) の列に平均から 4 標準偏差の外側のデータムが含まれていないことを確認します。 - AM列とVS列(それぞれ自動/手動とV/ストレートエンジン)に0と1のみが含まれていること - 各行には最大2つのNAが含まれます - 各行は、"mpg"、"am"、および "wt" 列の間で一意です。 - 各行のマハラノビス距離は、すべての距離の絶対偏差の中央値の10以内です(外れ値検出の場合)


上記の問題は次のように上から順番に条件を記述することで対応できる。


```{r}
library(dplyr)
library(assertr)

# 失敗するとデータフレームで結果が返される
# tryCatchはquartoで記述しているから必要である。本来は必要ない。
tryCatch({
  mtcars |> 
    verify(has_all_names("unko"))
}, error = \(e) print(e))

```

```{r}
# 成功した場合には処理を経たデータが返される
tryCatch({
  mtcars |> 
    verify(has_all_names("mpg", "vs", "am", "wt"))
}, error = \(e) print(e))

```


```{r}
# もともとやりたかった処理をまとめる
mtcars |> 
  verify(has_all_names("mpg", "vs", "am", "wt")) %>%
  verify(nrow(.) > 10) |> 
  verify(mpg > 0) |> 
  insist(within_n_sds(4), mpg) |>  # insistは「主張する」という意味である
  assert(in_set(0, 1), am, vs) |> 
  assert_rows(num_row_NAs, within_bounds(0, 2), everything()) |> 
  assert_rows(col_concat, is_uniq, mpg, am, wt) |> 
  insist_rows(maha_dist, within_n_mads(10), everything()) |> # マハラノビス距離と中央絶対偏差
  group_by(cyl) |> 
  summarise(avg.mpg = mean(mpg))



```



これを見ただけでもかなり使い方がイメージできる。

## enter assertr

次のような`verify`を使うことで計算が失敗するようになる。


```{r}
library(assertr)

tryCatch({
our.data |> 
  verify(mpg >= 0) |> 
  group_by(cyl) |> 
  summarise(avg.mpg = mean(mpg))

}, error = \(e) print(e))


```



`assert`関数を使うと次のように記述することができる。結果をみると行単位で評価結果を得られてることがわかる。


```{r}
tryCatch({

our.data |> 
  assert(within_bounds(0, Inf), mpg) |> 
  group_by(cyl) |> 
  summarise(avg.mpg = mean(mpg))
}, error = \(e) print(e))  
```


dplyrのselect関数が使える。


```{r}
library(assertr)

tryCatch({

our.data %>%
  assert(within_bounds(0,Inf, include.lower=FALSE), -mpg) %>%
  group_by(cyl) %>%
  summarise(avg.mpg=mean(mpg))
}, error = \(e) print(e))
```



## verify vs. assert

式を引数にとるか、述語関数やカラムを引数にとるかが異なる。
またverifyは引数全体に対する評価であり、全体を評価してから違反がないかを考えている。
このため行単位での問題の特定が出来ない。

## asserts predicates

述語関数には次のようなものがある。

- `not_na`はNAじゃないことを確認する
- `within_bounds`は数値が与えられた範囲に含まれているかを確認する
- `in_set`は集合のメンバーであるかどうかを確認する
- `is_uniq`はユニークかどうかを確認する

## success, error, and defect functions

成功・失敗時の処理の挙動を制御できる。

## Obligatory assertions

独立でないルールを適用することがある。


```{r}
tryCatch({

mtcars_without_am <- mtcars %>% 
  dplyr::select(-am)
mtcars_without_am %>% 
  verify(has_all_names("am", "vs"), error_fun = error_append) %>% 
  assert(in_set(0, 1), am, vs, error_fun = error_report)
}, error = \(e) print(e))
```


上記はverifyと関係ないところでエラーが出ている。
その場合には次のようにする。


```{r}
tryCatch({

mtcars_without_am <- mtcars %>% 
  dplyr::select(-am)
mtcars_without_am %>% 
  verify(has_all_names("am", "vs"), obligatory=TRUE, error_fun=error_append) %>% 
  assert(in_set(0, 1), am, vs, defect_fun=defect_report)
}, error = \(e) print(e))
```


## combining chains of assertions

パイプラインだと1つの処理が失敗するとそこから先の処理は実施されない。
一方で、複数のチェックをリポートしてもらいたいときがある。

```{r}
mtcars %>%
  chain_start %>%
  verify(nrow(mtcars) > 10) %>%
  verify(mpg > 0) %>%
  insist(within_n_sds(4), mpg) %>%
  assert(in_set(0,1), am, vs) %>%
  chain_end %>%
  group_by(cyl) %>%
  summarise(avg.mpg=mean(mpg))
```

## advanced: creating your own predicate generators for insist


```{r}
within_3_iqrs <- function(a_vector){
  the_median <- median(a_vector)
  the_iqr <- IQR(a_vector)
  within_bounds((the_median-the_iqr*3), (the_median+the_iqr*3))
}
mtcars %>%
  insist(within_3_iqrs, mpg) %>%
  group_by(cyl) %>%
  summarise(avg.mpg=mean(mpg))
```
