1 Parallel processing in purrr 1.1.0

  • blog

1.1 はじめに

  • map処理をマルチコアや分散システムにより拡張することができる
  • バージョンは1.1.0以降が必要である
  • in_parallel関数を使うことで、mapの処理を並列化することができる
  • mirai::deamonでコア数を調整する
Code 
packageVersion("purrr")
#> [1] '1.1.0'

1.2 The power of in_parallel()

まずは結果が同じになることを確認する。

Code 
library(purrr)
library(mirai)
#> Warning: パッケージ 'mirai' はバージョン 4.5.1 の R の下で造られました

# Set up parallel processing (6 background processes)
daemons(6)
#> [1] 6

# Sequential version
x <- mtcars |> map_dbl(\(x) mean(x))

# paralle version
y <- mtcars |> map_dbl(in_parallel(\(x) mean(x)))
daemons(0)
#> [1] 0


identical(x, y)
#> [1] TRUE

1.3 Real-world example: parallel model fitting

パラレル化したときには, オブジェクトは明示的に渡す必要があるし, ライブラリは明示的にロードする必要がある

Code 
library(purrr)
library(mirai)

# Set up max parallel processes
cores <- as.integer(Sys.getenv("NUMBER_OF_PROCESSORS"))
daemons(cores)
#> [1] 12

# Define a slow model fitting function
slow_lm <- function(formula, data) {
  Sys.sleep(5)  # Simulate computational complexity
  lm(formula, data)
}

# Fit models to different subsets of data in parallel
models <- mtcars |>
  split(mtcars$cyl) |>
  # printは効かない
  imap(in_parallel(\(df, idx) {print(idx); slow_lm(mpg ~ wt + hp, data = df)}, slow_lm = slow_lm))

# Extract R-squared values
models |> 
  map(summary) |> 
  map_dbl("r.squared")
#>         4         6         8 
#> 0.6807065 0.5889239 0.4970692
#>         4         6         8 
#> 0.6807065 0.5889239 0.4970692

daemons(0)
#> [1] 0

1.4 When to use parallel processing

  • パラレルが有効であるとき

  • 1イテレーションに, 100マイクロ秒から1ミリ秒以上かかる

  • CPU集約的な計算を行っている

  • I/Oバウンドな操作で, 並列性の恩恵を受けることができる

  • プロセス間で渡されるデータが過度に大きくない

  • また, deamonsはモジュールなどで呼び出さず, 必ずユーザーに呼び出させる

# Parallel processing in purrr 1.1.0

- [blog](https://www.tidyverse.org/blog/2025/07/purrr-1-1-0-parallel/)

## はじめに

- `map`処理をマルチコアや分散システムにより拡張することができる
- バージョンは1.1.0以降が必要である
- `in_parallel`関数を使うことで、`map`の処理を並列化することができる
- `mirai::deamon`でコア数を調整する

```{r}
packageVersion("purrr")
```

## The power of `in_parallel()`

まずは結果が同じになることを確認する。

```{r}
library(purrr)
library(mirai)

# Set up parallel processing (6 background processes)
daemons(6)

# Sequential version
x <- mtcars |> map_dbl(\(x) mean(x))

# paralle version
y <- mtcars |> map_dbl(in_parallel(\(x) mean(x)))
daemons(0)


identical(x, y)
```

## Real-world example: parallel model fitting

パラレル化したときには, オブジェクトは明示的に渡す必要があるし, ライブラリは明示的にロードする必要がある


```{r}
library(purrr)
library(mirai)

# Set up max parallel processes
cores <- as.integer(Sys.getenv("NUMBER_OF_PROCESSORS"))
daemons(cores)

# Define a slow model fitting function
slow_lm <- function(formula, data) {
  Sys.sleep(5)  # Simulate computational complexity
  lm(formula, data)
}

# Fit models to different subsets of data in parallel
models <- mtcars |>
  split(mtcars$cyl) |>
  # printは効かない
  imap(in_parallel(\(df, idx) {print(idx); slow_lm(mpg ~ wt + hp, data = df)}, slow_lm = slow_lm))

# Extract R-squared values
models |> 
  map(summary) |> 
  map_dbl("r.squared")
#>         4         6         8 
#> 0.6807065 0.5889239 0.4970692

daemons(0)

```

## When to use parallel processing

- パラレルが有効であるとき
- 1イテレーションに, 100マイクロ秒から1ミリ秒以上かかる
- CPU集約的な計算を行っている
- I/Oバウンドな操作で, 並列性の恩恵を受けることができる
- プロセス間で渡されるデータが過度に大きくない

- また, `deamons`はモジュールなどで呼び出さず, 必ずユーザーに呼び出させる
