2 Getting the data

ファイルサイズが巨大なデータとして, 4138万行を有するデータセットを用意する. 準備したファイルは9GBであり,　すべてのデータをメモリに読み込みたいくないデータとしては、 十便な大きさを有している. read_csvが通常使われるが, ここではarrow::open_datasetを使う.

```{r}
dir.create(here(cur_dir, "data"), showWarnings = FALSE)

curl::multi_download(
  "https://r4ds.s3.us-west-2.amazonaws.com/seattle-library-checkouts.csv",
  here(cur_dir, "data/seattle-library-checkouts.csv"),
  resume = TRUE
)

```

3 Opening dataset

4 The Parquet format

4.1 Advantages

  • Parquetは効率的なエンコードでファイルサイズを小さくする
  • CSVに持たせることが出来ない情報を持たせることができる
  • カラム指向である
  • チャンク化されていても,異なるチャンクを同時扱うことが出来る

4.2 Partitioning

  • データが巨大になるほど1つのファイルで管理するのが困難となる
  • チャンク化は20MB以下になることと2GB以上になることを避ける
  • ChekoutYearを軸にチャンク化をおこなう
  • ISBNがint64やnullだと出力が出来ないのでここでは文字列として出力する
Code 
pq_path <- here(cur_dir, "data/seattle-library-checkouts")
seattle_csv |> 
    group_by(CheckoutYear) |> 
    write_dataset(path = pq_path, format = "parquet")

4.4 Performance

データが絞られるほどCSVよりも高速に動作する.

4.5 Advance

parquetはカラム指向なので同じカラム指向のリレーショナルデータベースであるDuckDBと非常に相性がよい. DuckDBはカラム指向でデータの高速処理を指向して開発されたデータベースである. サーバーレスで動作する.

Code 
seattle_pq |> 
  to_duckdb() |>
  filter(CheckoutYear >= 2008, MaterialType == "BOOK") |>
  group_by(CheckoutYear) |>
  summarize(TotalCheckouts = sum(Checkouts)) |>
  arrange(desc(CheckoutYear)) |>
  collect() |> 
  system.time()
